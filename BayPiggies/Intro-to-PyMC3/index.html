What is Bayesian Statistics?¶

  • Frequentist Statistics is a method of statistical inference in which conclusions from data is obtained by emphasizing the frequency or proportion of the data. The resulting information is a distribution for some event. The resulting calculations are a single value, point estimate.
  • Bayesian Statistics is a method of statistical inference in which Bayes’ theorem is used to update the probability for a hypothesis as more evidence or information becomes available. The resulting calculationscalculations are a distribution for some event.

Why hasn't Bayesian Statistics been an industry-wide recommendation?¶

Similar to the growth of Machine Learning & Artificial Intelligence, Bayesian Statistics had set-backs and recent growth due to the following:

  • computationally challenging
  • conceptually challenging to work
  • Algorithms advancements like the Metropolis or No-U-Turn Sampler are more recently relevant to for the advancement of the field itself

Note: There is a consultancy group called PyMC3 Labs PyMC3 labs to help answer tough Business questions that require leveraging Bayesian Statistics.

Brief Introduction to Probability¶

Probabilities:

  • are in the [0, 1] Interval
  • follow the product rule $p(A,B) = p(A|B)p(B)$
    • $p(A, B)$ expression represents the joint probability of A and B
    • $p(A|B)$ expression is used to indicate a conditional probability of A and b
  • Independent when $p(A) = p(B)$

The Main Takeway - Bayes Theorem¶

Bayes' Theorem

$$p(\theta | y) = \tfrac{p(y|\theta)p(\theta)}{p(y)}$$

Note: Hypothesis: $\theta$ & $y$ is data

where

  • $p(\theta)$: Prior
  • $p(y | \theta)$: Likelihood
  • $p(\theta |y)$: Posterior
  • $p(y)$: Marginal Likelihood (evidence)

Getting Started with PyMC3¶

What is PyMC3?¶

PyMC3 is a Python package for Bayesian statistical modeling and Probabilistic Machine Learning. It's associated to one of several Probabilistic Programming Languages (PPL) which focuses on advanced Markov chain Monte Carlo (MCMC) and variational fitting algorithms, and more.

Some features of PyMC3 are the following:

  • Intuitive model specification syntax
  • Uses different Sampling Algorithms
    • Hamiltonian Monte Carlo Markov Chain
    • No U-Turn Sampler (NUTS)
  • Built on top of Theano (Aesara)
    • Theano features
    • Leverage dynamic C compilation for optimized computation
    • Automatic Differentiation

      Note: Theano is now "Aesara". See "The Future of PyMC3, or: Theano is Dead, Long Live Theano " for historical background & next steps

PyMC3 Example¶

Procedure would be:

  1. Instantiate Object
    • Create PyMC3 object w/with statement
  2. Inference
    • Calculate output distribution
  3. Predict & Evaluate

1. Instantiate Object¶

In [3]:
import pymc3 as pm
import numpy as np

p = 2/3 #probability
with pm.Model() as example_model:
    theta = pm.Uniform('theta', lower=0, upper=1) # prior variable

data = np.array([1,1,0]) #Observed data

Note: We assume that a Model example_model is not static. If you need to add additional variables, transformations, or more, you would need to re-instantiate the model with a with statement again

2.Inference¶

We now perform inference to approximate the posterior distribution.

The options for this inference is either by:

  • sampling
  • variational inference
In [ ]:
with example_model:
    obs=pm.Bernoulli("obs", p, observed=data) #input the observations
#     step=pm.Metropolis() # Proceed with sampling using Metropolis sampler instead of default, NUTS.
    trace = pm.sample(draws = 20_000, return_inferencedata=True)

We just did the following:

  • Instantiate a model called example_model
  • Set Prior (theta) as a Uniform Distribution w/ range [0,1]

  • Set the Likelihood (obs) as a Bernoulli Distribution w/probability 2/3.

  • Lastly, use PyMC3's pm.sample function to create a trace with 20,000 drawn samples from the posterior distribution

ArviZ is a Python package for exploratory analysis of Bayesian models. Includes functions for posterior analysis, data storage, sample diagnostics, model checking, and comparison.

The goal is to provide backend-agnostic tools for diagnostics and visualizations of Bayesian inference in Python
